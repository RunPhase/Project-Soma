02 — 계층별 상세 구성
수집 계층
데이터 수집
압력센서 × N개 → Arduino → pyserial
웹캠 → OpenCV 캡처 (30fps)
Serial 보드레이트 115200 권장
⚠ 협의 필요
유선 USB vs 무선(BLE) 결정 필요. MVP는 유선 추천.
분석 계층 (일부 내 파트)
멀티스레드 분석
Thread 1 (내 파트) — MediaPipe Pose, PnP head pose, 캘리브레이션
Thread 2 (팀원) — 압력 raw → 좌우 분포, 편중도 계산
Main Thread (팀원) — 공유상태 병합, posture_state 판정, ZeroMQ 전송
공유상태: threading.Lock 보호 dict
UI 계층 (내 파트)
Unity 렌더링
ZeroMQ SUB → JSON 파싱 → Humanoid 본 제어
Neck / Head: neck_angle + head_pitch
Shoulder: shoulder_tilt
PiP: RenderTexture → UI RawImage (우측 하단)
⚠ 사전 검증 필요
NetMQ Unity 메인스레드 제약 확인. TCP 직접 구현이 더 안전할 수 있음.
백엔드 계층 (팀원)
저장 + Web UI
FastAPI + PostgreSQL (시계열 자세 데이터)
캘리브레이션 baseline 저장/불러오기
Web 대시보드: 누적 피로도, 자세 이력 그래프
일/주간 리포트 생성
⚠ 중요
실시간 경로에 백엔드 개입 금지. 저장은 반드시 비동기.

04 — 캘리브레이션 흐름
초기 캘리브레이션
① UI: "올바른 자세로 앉으세요" 안내
② Thread 1: 3초간 측정 → 평균값 산출
③ baseline 로컬 저장 (json) + 백엔드 저장
④ 이후 판정은 baseline 대비 편차로 수행
앱 재시작 시 백엔드에서 baseline 불러오기
재캘리브레이션 트리거
단축키 또는 UI 버튼으로 언제든 재실행
의자 높이 변경 / 자리 이동 시 필수
캘리브레이션 중에는 판정 일시 중단
⚠ 협의 필요
캘리브레이션 UI 담당 확정 필요. 트리거 방식 (키보드 단축키 vs 버튼) 결정 필요.

06 — posture_state 정의 (초안)
State 캠 조건 압력 조건 비고
GOOD 정상 범위 균등 분포 —
BAD_NECK neck편차>15° AND pitch편차>10° AND face+15% — 3조건 AND, 3초 지속
BAD_SHOULDER shoulder_tilt_ratio 편차 > 0.1 — 3초 지속
BAD_WEIGHT — 좌우 압력비 > 1.4 3초 지속
BAD_COMPLEX shoulder_tilt 편차 > 0.1 압력 편중 > 1.4x 척추 측만 의심
ABSENT 랜드마크 미감지 전체 압력 ≈ 0 자리 이탈
CALIBRATING — — 캘리브레이션 중, 판정 중단
⚠  임계값은 모두 초안 — 캘리브레이션 실험 후 튜닝 필요  ·  posture_state 항목은 팀 전체 협의 필요
